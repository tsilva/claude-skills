# README Generator

Create stunning, modern README files that make projects stand out.

## Overview

Generates professional README.md files following GitHub best practices and modern design patterns. Analyzes your codebase to create comprehensive, visually appealing documentation.

### Logo Requirement

Every README generated by this skill includes a logo. The workflow:
1. **Check for existing logo** - look for `logo.png` at repo root
2. **Generate if missing** - use OpenRouter image models to create one
3. **Embed in hero section** - reference `logo.png` with proper alt text

## Features

### Visual Design
- **Centered hero section** with logo, title, and tagline
- **Badge collection** - build status, version, license, downloads, etc.
- **Organized sections** with clear visual hierarchy
- **Syntax-highlighted code blocks** with copy-friendly examples
- **Tables** for feature comparisons and API references

### Content Generation
- **Smart project analysis** - auto-detects project type, language, and framework
- **Installation instructions** - npm, pip, cargo, go, etc. based on detected package manager
- **Usage examples** - real code snippets from your codebase
- **API documentation** - extracted from code comments and type definitions
- **Contributing guidelines** - standard open-source contribution workflow
- **License detection** - reads and links to your LICENSE file

### Modern Patterns
- **Collapsible sections** for detailed content using `<details>` tags
- **Quick start guides** - get users running in under 2 minutes
- **Feature highlights** with icons or emojis (when appropriate)
- **Responsive images** that scale properly on all devices

## Logo Generation (Mandatory)

Every README must have a logo. Follow this 2-step process:

### Step 1: Check for Existing Logo

Look for `logo.png` at the root of the repo (same directory as README.md). If found, use it and skip to the README template.

### Step 2: Generate Logo (if none exists)

Use the OpenRouter skill with an image generation model:

```bash
python plugins/openrouter/skills/openrouter/scripts/openrouter_client.py image \
  "google/gemini-3-pro-image-preview" \
  "A [STYLE] logo for [PROJECT_NAME]: [VISUAL_METAPHOR]. [COMPOSITION_DETAILS]. Clean vector style, centered composition, no text, suitable for GitHub README, dark and light mode compatible." \
  --output /absolute/path/to/repo/logo.png
```

Save as `logo.png` at the root of the repo.

### Recommended Image Models

| Model | Best For |
|-------|----------|
| `google/gemini-3-pro-image-preview` | Quick iterations, good quality |
| `black-forest-labs/flux.2-pro` | Highest quality final logos |
| `black-forest-labs/flux-1.1-pro` | Balanced quality and speed |

## Logo Design Philosophy

### Core Principles

1. **Understand the project's core purpose** - what problem does it solve?
2. **Find visual metaphors** - represent the project's action, not just its name
3. **Keep it simple** - one clear concept, no clutter
4. **Make it distinctive** - avoid generic tech imagery (gears, circuits, clouds)

### Logo Concept Examples by Project Type

| Project Type | Example Concept |
|--------------|-----------------|
| CLI file converter | Document folding/morphing, origami style |
| Task runner | Sprinter or relay baton with motion lines |
| HTTP client library | Messenger pigeon carrying a letter |
| Validation library | Wax seal or approval stamp |
| Linter | Elegant lint brush sweeping dust |
| Test framework | Test tube with checkmark bubbles |
| Dashboard | Minimalist mission control desk |
| Analytics | Detective magnifying glass revealing patterns |

### Prompt Template

```
"A [STYLE] logo for [PROJECT_NAME]: [VISUAL_METAPHOR].
[COMPOSITION_DETAILS]. Clean vector style, centered composition,
no text, suitable for GitHub README, dark and light mode compatible."
```

**Example for a task queue library:**
```
"A minimalist logo for TaskFlow: A flowing river of geometric shapes
moving through a funnel. Gradient blues and teals, clean vector style,
centered composition, no text, dark and light mode compatible."
```

### Technical Requirements (always include)

- `"no text"` - prevents unreadable small text at logo sizes
- `"centered composition"` - works well in README headers
- `"dark and light mode compatible"` - readable on both GitHub themes
- `"clean vector style"` or `"minimalist"` - crisp scaling

## Workflow

1. **Analyze project** - scan for package.json, Cargo.toml, pyproject.toml, go.mod, etc.
2. **Detect project type** - CLI, library, web app, API, etc.
3. **Extract metadata** - name, description, version, author, license
4. **Check for existing logo** - look for `logo.png` at repo root
5. **Generate logo if missing** - use OpenRouter skill to create a memorable logo, save as `logo.png`
6. **Generate README.md** - create or update the file with the logo in hero section

```markdown
Creates a new Thing instance.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `timeout` | `int` | `30` | Request timeout in seconds |
```

## Best Practices Enforced

1. **Scannable content** - users should find what they need in seconds
2. **Copy-paste ready** - all code examples should work as-is
3. **Progressive disclosure** - basic info first, details in expandable sections
4. **Visual hierarchy** - clear headings, consistent formatting
5. **Mobile-friendly** - readable on GitHub mobile app
6. **Accessible** - alt text for images, semantic markup
